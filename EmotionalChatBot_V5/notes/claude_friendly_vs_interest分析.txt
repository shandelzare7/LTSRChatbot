## 1. friendly vs interest 的语义区别

| 维度 | friendly | interest |
|---|---|---|
| **定义** | 用户对**我（AI）**释放的友好/亲近信号 | 用户对**话题/对话本身**的投入兴趣度 |
| **语义指向** | 关系层面：「我喜欢你」 | 内容层面：「这个话题有意思」 |
| **典型高分例** | "你真好笑哈哈"、"谢谢你陪我聊" | "等等，这个展开说说"、"然后呢？"、长段认真回复 |
| **典型低分例** | 冷淡但在认真讨论技术问题 | "好的好的"（敷衍但语气不差） |
| **与冲量的因果关系** | 间接：对你友好 ≠ 想继续聊 | 直接：对话题感兴趣 = 想继续聊 |

**关键洞察：** friendly 和 interest 可以**完全解耦**：
- **高 friendly 低 interest**："你人真好，但我要去忙了" → 不应恢复冲量
- **低 friendly 高 interest**：用户在争论但越辩越起劲 → 应该恢复冲量
- **高 friendly 高 interest**："天哪你说的太对了，继续！" → 强恢复
- **低 friendly 低 interest**："嗯" → 不恢复

## 2. 哪个更适合冲量恢复？

**interest 更适合，原因：**

冲量模型的核心语义是「对话还能持续多久」。决定对话能否持续的**直接因素**是用户对话题/对话本身的投入度（interest），而不是用户对 AI 的友好度（friendly）。

用 friendly 恢复冲量存在两个问题：

**问题一：虚假恢复。** 用户礼貌性的友好（"谢谢你"、"你说的对"）会触发冲量回弹，但这类话往往是**收束信号**——用户在客气地准备结束对话。此时恢复冲量会导致 AI 继续积极输出，与用户意图相悖。

**问题二：漏检恢复。** 用户在热烈讨论但语气不带友好色彩（质疑、辩论、追问）时，friendly 低，冲量不回弹。但此时用户明明兴趣很高，对话应该继续。

interest 则直接对准了因果链：
```
用户感兴趣 → 用户会继续输入 → 对话持续 → 冲量不该衰减
```

## 3. friendly 是否完全不参与？

**推荐方案：friendly 低程度参与（约 0.03~0.04），interest 为主力（约 0.10~0.12）。**

理由：

- 完全移除 friendly 过于极端。在真实对话中，「对你友好」虽不等于「想聊更久」，但确实有轻微正相关——对你有好感的人更可能愿意多聊几句。
- 但权重要大幅降低。目前 `0.12 * friendly` 太高，friendly 承担了 interest 应该承担的功能。
- interest 作为主通道（0.10~0.12）承担冲量恢复的核心驱动。

## 4. 推荐方案

### Detection scores 新增 interest

```python
def _default_scores() -> Dict[str, float]:
    return {
        "friendly": 0.0,
        "hostile": 0.0,
        "overstep": 0.0,
        "low_effort": 0.0,
        "confusion": 0.0,
        "interest": 0.0,    # 新增：用户对话题/对话的兴趣投入度
    }
```

Detection prompt 中 scores 说明新增：
```
- interest: 用户对当前话题/对话本身的兴趣投入度（追问、展开、长段回复、主动引入新话题 → 高；敷衍、转移、想结束 → 低）。注意：interest 与 friendly 独立，用户可以不友好但高兴趣（激烈辩论），也可以很友好但低兴趣（礼貌告别）。
```

### `_compute_momentum` 改造

```python
def _compute_momentum(
    turn: int,
    invest: float,
    interest: float = 0.0,      # 主通道
    friendly: float = 0.0,      # 辅助通道
    low_effort: float = 0.0,
) -> float:
    params = _invest_to_momentum_params(invest)
    momentum = _momentum_curve(turn, **params)

    # 用户信号调制：interest 为主，friendly 为辅
    boost = 0.12 * interest + 0.04 * friendly - 0.08 * low_effort
    momentum = _clip01(momentum + boost)

    # 用户敷衍时额外衰减
    if low_effort > 0.6:
        momentum *= 0.88

    return momentum
```

### 权重对比

| 信号 | 旧权重 | 新权重 | 变化 |
|---|---|---|---|
| friendly | 0.12 | 0.04 | 降为 1/3 |
| interest | — | 0.12 | 新增主通道 |
| low_effort | -0.08 | -0.08 | 不变 |

### 为什么是这组权重？

- **interest 0.12**：继承原 friendly 的主力位。当 interest=1.0 时冲量回弹 +0.12，足以抵消约 1.5 轮的自然衰减（bonding 阶段）。
- **friendly 0.04**：保留微弱正贡献。friendly=1.0 时仅 +0.04，不会因为客套话就大幅回弹，但在 interest 也高时能叠加出更强的恢复。
- **low_effort -0.08**：不变，语义清晰（敷衍 = 对话衰减），无需调整。

### interest 与 low_effort 的关系

可能有人问：interest 低 ≈ low_effort 高？不完全是。它们有重叠但不等价：
- low_effort 偏行为层面（短句、敷衍词、不接球）
- interest 偏认知层面（是否在认真思考话题）
- 用户可以短句但高兴趣（"真的？？？"、"等等"）
- 也可以长句但低兴趣（复制粘贴、机械重复）

两者独立计分，在冲量公式中各司其职，不会冗余。
